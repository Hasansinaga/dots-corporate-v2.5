## Reporting a Vulnerability

Kami sangat menghargai jika Anda melaporkan kerentanan keamanan yang Anda temukan. Untuk melaporkan kerentanan keamanan:

### 1. **Jangan buat issue publik**
Jangan posting kerentanan keamanan sebagai issue publik di GitHub.

### 2. **Email langsung ke tim keamanan**
Kirim email ke: `security@dots-corporate.com`

### 3. **Informasi yang diperlukan**
Sertakan informasi berikut dalam laporan Anda:
- Deskripsi kerentanan
- Langkah-langkah untuk mereproduksi
- Dampak potensial
- Saran perbaikan (jika ada)

### 4. **Timeline response**
- **24 jam**: Konfirmasi penerimaan laporan
- **72 jam**: Update status investigasi
- **7 hari**: Rencana perbaikan (jika dikonfirmasi)
- **30 hari**: Patch tersedia (jika memungkinkan)

## Security Best Practices

### Untuk Developers
- Jangan commit credentials atau API keys ke repository
- Gunakan environment variables untuk konfigurasi sensitif
- Update dependencies secara berkala
- Ikuti secure coding practices
- Lakukan security review sebelum merge PR

### Untuk Users
- Update aplikasi ke versi terbaru
- Jangan share credentials dengan siapapun
- Laporkan aktivitas mencurigakan
- Gunakan password yang kuat

## Security Updates

Kami akan mengumumkan security updates melalui:
- GitHub Security Advisories
- Release notes
- Email notification (untuk critical vulnerabilities)

## Responsible Disclosure

Kami mendukung responsible disclosure dan akan:
- Mengakui kontribusi keamanan Anda
- Bekerja sama untuk memperbaiki kerentanan
- Memberikan credit di security advisories
- Tidak mengambil tindakan hukum terhadap security researchers yang mengikuti kebijakan ini
